# 从 Toy 到可用：后续演进路线

这份文档记录已完成的版本，并规划 V8 及之后要解决的问题。每一步都只引入最少的变化，保持独立可验证。

## 已完成的版本

| 版本 | 解决的问题 | 为什么重要 |
|------|-----------|-----------|
| V1 单轮对话 | agent 的最小闭环能跑起来 | 奠基。不理解 V1 就看不懂后续 |
| V2 多轮交互 | 进程内持续对话 | 体验从"命令行工具"变为"对话助手" |
| V3 流式 + 消息结构 | 流式输出 + 标准 user/assistant 消息 | agent 从"黑箱"变"透明"，消息结构正确 |
| V4 会话持久化 | 对话历史可保存/恢复 | agent 从"一次性工具"变为"有记忆的助手" |
| V5 补丁式编辑 | 精确文本替换，不重写整个文件 | 节省 token、减少意外改动 |
| V6 命令安全 | 白名单 + 危险模式拦截 + 超时 | agent 不能执行危险命令 |
| V7 自动验证闭环 | 修改代码后必须验证通过才能 finish | agent 从"裸改"到"改验一体" |

## V8+（待规划）

**可能方向**：
- 文件级 diff 展示
- 错误重试上限
- 更精细的验证命令判断

## 学习建议

每个版本都可以对照着上一个版本的代码读：
- **增量很小**：每次改动通常只涉及 1-2 个文件。
- **目的明确**：每个版本解决一个具体问题。
- **独立可跑**：版本目录里的代码永远是完整、可执行的。
